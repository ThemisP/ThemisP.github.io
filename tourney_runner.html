Tourney Runner
Web APP for game tournament creation

Introduction

This is a project undertaken with a team of 6 and I attempted the role of lead programmer. The idea is to create a front-end together with a back-end that can support an application with the function of creating tournaments between users for various reasons, wether to compete at a friendly 8 people pool tournament or a big football tournament between 32 or more teams.

Technologies: Server developed in Java with SpringBoot Framework and an h2 integrated database with a RestAPI for communication between server and client. The server provides the client with a web application resource written in HTML, CSS and Javascript with JQuery-Mobile as the main library for the APP.

Front-End

At the beginning of the project, the minimum viable product (MVP) was identified so as to try and stick to a strict plan, develop something that works and if time permitted, add extra functionality. The MVP plan was to have the most common tournament structure added which is knockouts tree with users that can easily create tournaments and add rules such as max game points. Also have a very user friendly interface with users able to simply join tournaments with invitations or codes.

  • There were many plans and ideas in the team at the start of the development, therefore we started templating some of the ideas and testing user interfaces in an attempt to find the most user friendly and aesthetically pleasing layout. Such an early stage plan is shown in Figure 1.
  • The application has a very simple use case diagram Figure 2 that shows how simple and effective we planned the application to be. Just as explained above we aimed for simplicity and ease of use.
  • Technical: As mentioned in the introduction for the front-end development, HTML, CSS and Javascript was used with the JQuery-Mobile library. The specific library is a single page application format that provides good widgets and structures that allow for development of a website in an application style capable of running smoothly in a mobile phone. Code for the front end is located in the Git repository (button next to the Introduction title) and inside the src/main/resources folder.
